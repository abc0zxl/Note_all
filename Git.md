# Git


**Git**：他是一个开源的**分布式版本控制系统**，**分布式版本控制工具**



## GitHub

Git和GitHub，就是Git的**远程**代码托管中心，简称**远程库**，是一个代码托管平台

* **push**：代码推送
* **pull**：代码拉取
* **Clong**：代码克隆，down代码
* **SSH**：免密登录

**Gitee**：国内的一个类似于GitHub的平台

**GitLab**： 基于局域网的代码托管平台，



**版本控制**：

* 记录文件内容变化
* 记录文件修改历史记录
* 获取文件的历史版本
* 配合多人协作的同一个文件

#### 集中式版本控制工具

例如有：CVS，SVN，VSS

1.有一个集中的管理服务器

2.都把代码放到这个服务器

3.在服务器实现版本控制

4.管理员可以管理员工的开发权限

**坏处**：

* 中央服务器太单一，如果故障所有人都不可以用


## 分布式版本控制工具

例如：Git

**远程库**：GitHub，Gitee等

**优点**：没有固定的单一服务器，损坏容错更大

* 可以往**远程库**发送代码
* 版本控制是在本地进行的
* 远程库坏了可以用其他客户端的本地仓库恢复（因为每次提取都是一次完整的克隆）


## Git分区

分为三个区：本地库，暂存区，工作区。

![image.png](/assets/5c6d0dec-42a5-4e04-a0da-9076e23a30d5.png)

**工作区**：代码目录，也指存放的位置


**暂存区**：添加到一个临时存储区的代码，这个特点是还可以修改（删除，修改）


**本地库**：一旦将**暂存区**代码放到**本地库**，这种情况就会生成**历史版本**，就删不掉了。再修改也只是，再多了一个历史版本，原版本一直在。

**用户签名**：首次安装Git时设置的，用于声明提交的身份，**不然无法提交**


**追踪**：本地文件的两种状态，**已追踪**，**未追踪**

![image.png](/assets/b023cfc1-d631-40f9-9090-b6d1f8d0e793.png)

![image.png](/assets/1b1e8158-44ca-42bd-b94f-37e155612de8.png)

**常用命令**：

* **初始化本地库**：git init
  本地库可以有多个（不同目录即可），切换不同的目录提交，互不干扰。
* **查看本地库**：git status
* **添加到暂存区**：git add file_name

  ![image.png](/assets/15d573d4-956f-486c-a0cc-59d928d9a195.png)
* **提交到本地库**：git commit -m"log_name" file_name
* **查看版本信息**：git reflog

  ![image.png](/assets/3ded5298-6b09-4b63-b99a-ad80c0024285.png)

  **更详尽的日志信息**：git log

  ![image.png](/assets/f74402eb-9018-4738-a8aa-51b1af57f7f3.png)
* **修改后的文件**：vim file_name。这个状态表示文件已被修改，且在目录区

  ![image.png](/assets/d6cf13e5-acc0-4b04-a71d-d1d6475f3f11.png)

  这个提交的**log_name**字段就是表示的是一个名称，比如可以名命为，first_file,second_file,three_file

  ![image.png](/assets/0ed38322-ed27-4128-b72e-7cd2b9c86d1e.png)
* **版本指针**：获取这个文件时，能获取的版本
* **版本穿梭**：git reset --hard 版本号

  表示想获取其他版本的文件

  流程：获取想要的版本的版本名（log_name）,然后获取版本号，运行指令，更改版本 指针，查看

  ![image.png](/assets/94d1b58e-be45-462a-beef-9e04647d14ae.png)
* **分支**：head指向的是**分支**，例如：master，main。版本跃迁时，修改的是master

  ![image.png](/assets/6a3e3dc5-00dd-4f33-a80d-0208f7f0190f.png)
