# 关键词解释

## Tool

他是在智能体不能处理特殊的功能的时候，可以调用的一个工具，这个工具是自己在电脑上写的。

**给大模型的一个可以感知的外部环境（并不是大模型来调用他）**

**原理**：

* 我们在输入框输入需求（平台）
* 这个需求会发送到大模型（同时会携带上有哪些工具）
* 大模型处理完后会发送到平台，并给平台将数要调用哪些工具
* 调用完工具后，数据全部在平台上，然后再次发送给大模型，大模型处理完后就会给平台
* 最后将处理好的数据给到用户。

## MCP

Model Context Protocol统一接入标准

**作用**：为了解决下面的各个平台的接入标准不同的问题。

**原因**：上面说到的**Tool**，他是自己编写的一个功能，大模型还不能感知到它，自己的平台也不能调用到它。

**痛点**：发过去的请求**有严格的格式要求**，大模型返回的JSON响应**也有固定的格式**。

* 各个平台的接入规范都不一样




## Agent

**自主规划，调用工具**

**原因**：有一些复杂的问题，需要同时调用多个工具

![image.png](/assets/4e63c408-8216-404f-89a7-237f08f68112.png)

**作用**：他是一个调度中心，能够和大模型主动配合，自主完成一系列循环。


## Agent skill

**原因**：如果想让大模型，按照你的思路来思考，给她分配角色和要思考哪些东西。让每次回答的内容在你的预想范围内。

* 每次都要给大模型敲一大段话，非常麻烦

**作用**：用一个文件，集中管理，提前设定好要让大模型要干的事情。

* 目标
* 执行步骤
* 判断规则
* 输出格式
* 示例：用户问题，定位工具，天气工具，最终输出。

**注意**：必须按照Agent Skill的命名规范来设定名字。
