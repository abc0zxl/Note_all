# 渐进式

意思是vue可以像搭积木一样，从最小的核心功能开始，逐步扩展到完整的大型应用，不必一开始就把所有东西用上。

1.前端开发不是只有**大学企业应用**一种场景，而是从**小需求**到**大工程**全场景。

* **小场景**：点击按钮+表单验证
* **中场景**：个人博客，小型管理页面，包含路由，
* **大场景**：路由+状态管理+工程化+ui组件库

2.**优点**：React哪怕写一个hello world也要引入React+ReactDOM

* 在早期的前端框架中，用的都是全量框架，要么**全用**，要么**不用**
* 在编写vue项目的时候就能**只实现核心**，周边**生态**是可选。**按需加**
* **对开发者来说**：降低了学习门槛，可以**学一点用一点**，

3.**React**：强调**一套方案走到底**，牺牲了小场景适配以及新手的友好性。

# 响应式

是一个**数据驱动视图**的机制，修改数据是，vue回自动检测到数据变化。并将数据渲染到dom部分

**地位**：ref是vue3中最基础，最常用的API之一

**原理**：ref(0)返回的**不是普通数字**，他返回的是一个**响应式包装对象**

**作用**：

* 支持**数字，字符串，bool，复杂对象，数组**
* 通过包装变量让其具备**可监听能力**，修改时Vue能立刻感知并**更新视图**

![image.png](/assets/17b45168-762e-4020-9a39-509200ed7d5f.png)

**使用方法**：**必须**需要使用变量的value方法打开它

# 工程化开发（脚手架）

这里讲解开发vue的方式

1.**传统模式**：通过html**引入vue核心包**，开发Vue

2.**工程化开发模式**：

* webpack：**他是一个打包工具，处理代码的工具**，没有界面
* **作用**：将例如**ts,less等**的各个文件，**转换为可执行的js,css文件**
* 如果vue项目没有集成webpack，vite就需要构建build**打包成静态文件**，然后再用serve**运行这个静态文件**

3.**判断vue是否集成了webpack或者vite**

* 是否有：“serve”:"vue-cli-service serve"或者“build”:"vue-cli-service build"
* 是否有**vue.config.js**或者**webpack.config.js**

## Vue CLI

他是Vue官方提供的一个全局命令工具

**作用**：快速**创建一个**开发Vue项目的**标准化基础架子**

**好处**：

* 开箱即用
* 内置babel等工具
* 标准化

#### 使用步骤

1.**安装CLI**：npm i @vue/cli -g

2.**查看Vue版本（这个随意）**：vue --version

3.**创建项目架子（vue项目）**：vue create project-name

4.**启动项目**：npm run serve
