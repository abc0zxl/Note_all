# 渐进式

意思是vue可以像搭积木一样，从最小的核心功能开始，逐步扩展到完整的大型应用，不必一开始就把所有东西用上。

1.前端开发不是只有**大学企业应用**一种场景，而是从**小需求**到**大工程**全场景。

* **小场景**：点击按钮+表单验证
* **中场景**：个人博客，小型管理页面，包含路由，
* **大场景**：路由+状态管理+工程化+ui组件库

2.**优点**：React哪怕写一个hello world也要引入React+ReactDOM

* 在早期的前端框架中，用的都是全量框架，要么**全用**，要么**不用**
* 在编写vue项目的时候就能**只实现核心**，周边**生态**是可选。**按需加**
* **对开发者来说**：降低了学习门槛，可以**学一点用一点**，

3.**React**：强调**一套方案走到底**，牺牲了小场景适配以及新手的友好性。

# 响应式

是一个**数据驱动视图**的机制，修改数据是，vue回自动检测到数据变化。并将数据渲染到dom部分

**地位**：ref是vue3中最基础，最常用的API之一

**原理**：ref(0)返回的**不是普通数字**，他返回的是一个**响应式包装对象**

**作用**：

* 支持**数字，字符串，bool，复杂对象，数组**
* 通过包装变量让其具备**可监听能力**，修改时Vue能立刻感知并**更新视图**

**使用方法**：**必须**需要使用变量的value方法打开它

# 异步更新

当有两个语句放在一起有先后关系，**但是后面的语句需要前面的语句加载完才能执行**。

1.**异步**：在vue中即使有前后关系，这种依赖也会报错，因为vue是异步的，**后面的代码执行的时候，前面的代码不一定就执行完了**

2.**如图就会报错**：![image.png](/assets/4be0bf00-e614-4e4e-973b-0660533eb8b6.png)

### 解决方法$nextTick()

通过$nextTick()包裹这个事件，

**作用**：他会等DOM跟新完再去执行内部的方法。这样就不会报错了
