# Docker

**问题**：

* **环境问题**：不同电脑或者服务器上，环境配置不同，导致程序无法运行
* **依赖冲突**：缺少依赖，版本冲突，运行出错

**解决**：

* **打包方式**：程序和依赖环境一起打包，确保在任何环境都可以运行
* **镜像**：无需手动配置环境，提高开发效率

**工作原理**：

* 把程序和其运行环境依赖打包成镜像（盒子），实现环境一致性。

# 优势

1.**开发更轻松**

2.**部署更高效**

3.**容器之间互不影响**，也不会干扰系统

4.占用的资源更少

### 容器

他是镜像运行的实例

### Docker

负责容器的创建和管理

### Docker仓库

用于存储和分法镜像

* Docker Hub
* 阿里云镜像仓库

### 镜像

他是一个静态的，只读的打包模板，包含了程序运行所需的环境和依赖



# 应用场景

1.快速启动MongoDB，Redis，MySQL

2.**版本的控制和回滚**

3.他是DevOps标配

4.云原生和容器化部署


## Docker和虚拟机的区别

1.Docker只运行一个盒子程序

2.虚拟机模拟的是整个系统
